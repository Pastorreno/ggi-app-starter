# Executive Assistant Setup Plan for GGI Hub

## Executive Summary

If this were being run by an executive assistant, GGI Hub would be organized around one simple objective:

```text
Turn conversations into paid projects using two clear offers.
```

The two offers are:

1. Event Landing Pages — fast cash
2. Church Follow-Up Systems — recurring revenue

The website, GitHub repo, outreach, content, and client delivery should all support those two offers for the next 30 days.

---

## Primary Operating Rule

Do not build anything that does not help sell or deliver one of the two selected offers.

```text
Event pages create quick wins.
Church follow-up systems create monthly revenue.
```

# 4. Weekly Execution Rhythm

## Monday — Offer Focus

Pick one offer to push this week:

- Event pages, or
- Church follow-up system

## Tuesday — Content

Post one piece of content connected to that offer.

Examples:

- Churches: "Visitors should not fall through the cracks."
- Events: "Your event needs more than a flyer. It needs a page."

## Wednesday — Outreach

Send 10 direct messages or emails.

Targets:

- pastors
- ministry leaders
- event hosts
- nonprofit leaders
- small business owners

## Thursday — Follow-Up

Follow up with everyone who responded or opened the conversation.

## Friday — Proposal / Delivery

Send proposals, build pages, or review active client work.

# 6. Delivery Workflow

## Event Page Delivery

1. Client submits event details.
2. GGI Hub confirms package.
3. Collect deposit or full payment.
4. Build page.
5. Connect RSVP/registration form.
6. Send preview link.
7. Make one revision round.
8. Publish final page.
9. Send attendee export instructions.
10. Offer post-event follow-up system.

## Church Follow-Up Delivery

1. Discovery call.
2. Identify current follow-up process.
3. Map visitor intake.
4. Build basic tracker.
5. Build leader dashboard.
6. Set status labels.
7. Train point person.
8. Review first week of data.
9. Send weekly report.
10. Offer upgrade into onboarding/care/leadership tracking.
